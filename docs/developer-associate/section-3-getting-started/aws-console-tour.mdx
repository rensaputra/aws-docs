# Tour of the AWS Console & Services in AWS

In this section, I navigate through the AWS Console, including region selection and service accessibility.

## Key Takeaways

- Introduction to the AWS Console Home and its layout
- It is advised to choose a region that is geographically close to you for this course, because I'm in Sydney, I choose the Sydney region (ap-southeast-2) for the rest of the course.
- Overview of the console home, it has useful widgets like "Recently visited services" and "AWS service health" that provide quick access to frequently used services and updates on service status.
- Options for navigating services: browsing alphabetically, by category, or using the search bar.
- Explanation of **global services** like `Route 53` that don't require a region selection versus services like `EC2` that do.
- Emphasis on remaining in the same region throughout the course to avoid issues with resource availability.
- Introduction to AWS global infrastructure for checking [service availability by region](https://aws.amazon.com/about-aws/global-infrastructure/regional-product-services/).
